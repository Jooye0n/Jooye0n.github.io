## Chapter 1. 백앤드 개발이란 무엇인가?
---
### 1. 백엔드 개발이란
<img width="600" alt="image" src="https://github.com/user-attachments/assets/c2971893-c079-4c06-b5c2-f4ce881520f6">   
위 그림처럼 백엔드 개발은 웹 개발 내에 존재하는 개발 영역이다.  
여기서 웹은 우리가 흔히 볼수 있는 도메인의 가장 첫 부분 WWW(World Wide Web)에서 따온 말이다.   

### 2. 인터넷과 웹
<img width="600" alt="image" src="https://github.com/user-attachments/assets/73b70df1-ac70-448e-9823-43dd4a83d48a">  
인터넷과 웹은 비슷한 의미로 사용하곤 하지만 다른 의미를 가진 용어이다.  
인터넷은 전 세계에 연결된 네트워크 망 그 자체를 의미한다. 우리가 사용하는 PC나 기업에서 운영하는 서버 그리고 라우터나 스위치 같은 네트워크 장비들이 포함된 네트워크 망을 의미한다.  
반면 웹은 이 인터넷 망 위에 HTTP라는 프로토콜을 사용해서 동작하는 시스템일 뿐이다.  
즉 인터넷이 더 넓은 개념이라고 할 수 있다. 인터넷에는 HTTP 말고도 다른 프로토콜들도 많이 사용이 되고 있고, 이들은 우리가 웹이라고 부를 수 있는 것들은 아니다.  

### 3. 프런트엔드와 백엔드
<img width="600" alt="image" src="https://github.com/user-attachments/assets/57883b13-3271-4dbb-a737-b1b3eeb26a99">  
최초의 웹 개발은 굉장히 단순했다. HTTP 프로토콜을 사용해서 단순히 HTML 문서를 전달할 뿐이었다. 당연히 프런트 엔드와 백엔드 같은 구분이 필요 없었다.  
그러다 거기에서 조금 더 발전해서 지금 이 그림같이 프런트엔드와 백엔드라고 구분할 만한 게 생다.  
서비스가 복잡해지면서 단순히 HTML 문서만 주고받는 게 아니라, 서비스에 데이터를 저장하고 사용자마다 다른 화면이나 또 다른 데이터를 보여줘야 될 필요가 생겼기 때문이다.그에 따라서 이렇게 데이터베이스 같은 요소가 필요해졌다.  
데이터베이스에 데이터를 저장하고 사용자가 요청을 할 경우 그 저장된 데이터를 다시 사용자에게 응답으로 주는 로직 같은 걸 웹서버에다 작성하기 시작했다. 이렇게 사용자의 요청을 받아 데이터를 처리하기 위해 어떤 무언가 로직이 들어가는 안쪽 부분이 백엔드가 된다. 반대로 사용자가 볼 수 있는 부분 HTML 문서나 아니면 자바스크립트 같은 언어로 작성된 코드가 바로 프런트엔드에 해당하게 되었다. 그리고 한동안 많은 서비스들이 이 구조를 사용했다.  

<img width="600" alt="image" src="https://github.com/user-attachments/assets/e224be86-8469-4993-b7b2-3f654332a3d9">  
그런데 이 구조를 깰 만한 큰 변화가 하나 생겼다. 바로 스마트폰의 등장이다. 백엔드 서버는 웹 브라우저를 사용하는 사용자에게는 HTML 문서를 만들어서 줘야 했지만 스마트폰 사용자에게는 HTML 문서를 통째로 주는 게 좋은 방법이 아니었다. 왜냐하면 스마트폰 애플리케이션들은 각각의 ui가 이미 존재하는 경우가 많았기 때문이다.  
스마트폰 애플리케이션 ui에는 HTML 문서가 필요한 게 아니라 그 HTML 문서 내에 사용자마다 조금씩 다르게 들어가 있는 데이터만 필요하다. 만약 백엔드 서버가 HTML 문서를 그대로 줄 경우, 스마트폰 애플리케이션에서 그 HTML 문서 내에 있는 필요한 데이터들을 뽑아내는 로직을 따로 작성을 해줘야한다. 즉, 데이터 파싱이 필요하다.  
> 파싱은 꼭 이렇게 스마트폰 애플리케이션이 아니더라도 HTML 문서 같이 어떤 특정한 문자열이 있을 때 그 문자열 안에서 내가 사용하고 싶은 유의미한 데이터를 뽑아내는 걸 지칭한다.
